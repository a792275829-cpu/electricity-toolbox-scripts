# 电力脚本统一工具箱设计

## 目标

将现有六个工具整合到一个 Windows 桌面窗口中：

1. 导出上网电量
2. 电力交易分析
3. 电量汇总
4. 生成报告
5. 私有数据上传
6. 上传集团每日数据

用户通过左侧导航切换功能页面，不再频繁打开和关闭不同脚本。切换页面不应中断正在运行的任务，也不应清空其他页面的输入、日志或状态。

## 总体结构

新增一个 Python 3.11 Tkinter 主程序，采用左侧固定导航和右侧页面容器：

- 主窗口默认尺寸约为 1100 x 720，并允许缩放。
- 左侧显示六个功能入口，并突出显示当前页面。
- 右侧页面按需切换，页面实例在程序生命周期内保留。
- 底部提供统一状态栏，显示当前页面和后台任务概况。
- 在 `00_启动` 中新增统一启动批处理，原有六个启动入口继续保留。

不引入新的第三方界面框架，继续使用现有 Python 3.11 和 Tkinter，减少安装及兼容性风险。

## 页面设计

### 导出上网电量

- 日期输入，默认当天。
- 导出按钮。
- 登录或刷新登录状态按钮。
- 打开输出目录按钮。
- 独立运行日志和进度状态。
- 复用 `上网电量抓取/export_online_energy.py` 的核心函数。

### 电力交易分析

- 添加、移除和清空 Excel 文件。
- 选择输出目录。
- 批量生成 HTML 报告。
- 打开输出目录。
- 独立运行日志和进度状态。
- 复用 `电力工具脚本/电力交易分析工具/generate_electricity_report.py`。

### 电量汇总

- 选择原始数据文件夹。
- 选择输出 Excel 文件。
- 开始汇总。
- 打开输出目录。
- 独立运行日志和进度状态。
- 复用 `电力工具脚本/电量汇总工具/summarize_511_excel.py`，保留现有单元格数值和显示格式处理。

### 生成报告

- 报告日期。
- 三个 Excel 数据源和可选 Word 模板。
- 自动匹配文件。
- 抓取网站数据、生成 Word 报告选项。
- 打开输出目录。
- 独立运行日志和进度状态。
- 复用现有报告脚本及其子进程调用方式。

### 私有数据上传

- 选择复盘文件夹。
- 预览匹配文件。
- 确认后开始上传。
- 独立运行日志和进度状态。
- 直接调用现有 Node 脚本 `upload-private-data.mjs`，不再依赖弹出 PowerShell WinForms 主窗口。

### 上传集团每日数据

- 选择能销和省内 Excel 文件。
- 显示识别日期和文件类型。
- 开始上传。
- 登录或刷新登录状态。
- 可选择覆盖当天已上传数据。
- 独立运行日志和进度状态。
- 复用 `集团每日上传/upload_daily_report.py` 的上传逻辑。

## 任务隔离

- 每个页面只管理自己的输入、按钮、日志、进度和工作线程。
- Python 计算任务使用独立后台线程，避免阻塞主界面。
- Playwright、Node 和多步骤报告任务优先使用独立子进程，避免浏览器会话、工作目录和环境变量互相污染。
- 页面切换只改变显示页面，不销毁页面对象。
- 同一页面任务运行期间禁用该页面的执行按钮，其他页面仍可操作。
- 子进程使用对应工具目录作为工作目录，并显式启用 UTF-8 输出。
- 关闭主窗口时，如存在运行中的任务，要求用户确认；确认后终止由工具箱启动且仍在运行的子进程。

## 路径与兼容性

- 所有工具路径基于统一工具箱脚本自身位置计算，不依赖当前工作目录。
- 优先使用固定 Python 3.11 路径，找不到时再使用当前解释器或 `py -3.11`。
- 中文路径通过 `pathlib.Path`、参数列表和 PowerShell `LiteralPath` 处理，避免字符串拼接造成乱码或转义错误。
- 子进程设置 `PYTHONUTF8=1` 和 `PYTHONIOENCODING=utf-8`。
- 不改变现有配置、登录状态、输出目录和历史文件。

## 错误处理

- 启动前检查脚本、Python、Node 及必要目录是否存在。
- 输入文件或目录无效时在当前页面提示，不影响其他页面。
- 后台异常写入当前页面日志，并恢复该页面按钮状态。
- 子进程非零退出时显示退出码和关键日志。
- 上传类操作在真正执行前保留确认步骤。

## 验证

1. 对新程序和被调整模块执行 Python 语法编译检查。
2. 验证统一启动批处理能从任意当前目录启动工具箱。
3. 打开主窗口并依次切换六个页面，确认无额外主窗口弹出。
4. 验证每页文件选择、目录选择、日志和状态控件。
5. 启动一个可控的后台测试任务后切换页面，确认界面保持响应且状态不丢失。
6. 对不产生实际上传的功能执行冒烟测试；上传功能使用预览、参数校验或登录入口测试，避免误传生产数据。
7. 验证原六个启动入口仍然可用。

## 非目标

- 不重写各工具的业务计算和上传规则。
- 不删除原脚本、配置、登录状态或启动入口。
- 不增加自动更新、用户权限、多语言和远程服务。
- 不在测试中执行未经用户确认的生产数据上传。
