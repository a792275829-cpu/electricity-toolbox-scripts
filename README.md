# 电力工作工具箱 macOS 版

这是一个面向 macOS 的电力业务桌面工具集合。统一入口为 Tkinter 图形界面，当前集成 9 个工具：上网电量导出、市场表更新、广东电价预测、电力交易分析、电量汇总、生产经营报告生成、私有数据上传、集团每日数据上传和 WPS/KDocs 写入。

仓库公开地址：[a792275829-cpu/electricity-toolbox-scripts](https://github.com/a792275829-cpu/electricity-toolbox-scripts)

## 环境要求

- macOS 12 或更新版本
- Python 3.11 或更新版本
- Google Chrome，用于网页登录、上传和 WPS/KDocs 流程
- Node.js 18 或更新版本，仅“私有数据上传”需要
- 能访问相关业务系统的网络环境；如公司网络要求代理或 VPN，请先连接

项目使用根目录下的 `.venv` 作为 Python 环境。不要把系统 Python 或个人电脑上的绝对解释器路径写入项目配置。

## 首次安装

### 1. 获取项目

使用 Git 克隆：

```bash
git clone https://github.com/a792275829-cpu/electricity-toolbox-scripts.git
cd electricity-toolbox-scripts
```

也可以从 GitHub 下载 ZIP，解压后进入项目根目录。

### 2. 解除 macOS 下载限制

从网页下载的项目可能带有隔离属性。首次使用时在项目根目录执行：

```bash
xattr -dr com.apple.quarantine .
chmod +x setup_macos.command "电力工具箱.command" "00_启动/电力工具箱.command"
```

### 3. 自动安装依赖

在项目根目录运行：

```bash
./setup_macos.command
```

安装脚本会完成以下工作：

1. 创建项目专用 Python 环境 `.venv`。
2. 安装 `requirements.txt` 中的 Python 依赖。
3. 安装 Playwright Chromium。
4. 检测 Node.js/npm；如果存在，则安装私有数据上传工具的 Node 依赖。
5. 修复两个 `.command` 启动文件的执行权限。

如果暂时没有 Node.js，其他 Python 工具仍可使用，只有“私有数据上传”不可用。安装 Node.js 后执行：

```bash
cd "电力工具脚本/private-data-uploader-tool"
npm install
```

### 4. 手动安装命令

只有自动安装失败时才需要使用以下等价命令：

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -m playwright install chromium

cd "电力工具脚本/private-data-uploader-tool"
npm install
```

## 启动工具箱

在 Finder 中双击以下任一文件：

```text
电力工具箱.command
00_启动/电力工具箱.command
```

也可以在项目根目录从终端启动：

```bash
./电力工具箱.command
```

启动器会优先使用 `.venv/bin/python`。如果 `.venv` 或关键依赖缺失，会自动调用 `setup_macos.command`；初始化仍失败时才回退到系统 `python3`。

启动后默认显示“今日概览”：

- 左侧按“数据采集”“分析与报告”“上传与写入”分组。
- 工具页面首次打开时加载，切换页面不会清空当前状态。
- 右侧任务中心显示后台任务、运行状态和日志，并提供可用的取消操作。
- “运行环境诊断”会检查 Python、9 个工具模块和可选的 Node.js 依赖。

## 首次登录和本地配置

仓库只提交示例配置。真实账号、密码、Cookie、浏览器 profile、登录态和 WPS 配置均为本机文件，已由 `.gitignore` 忽略，不会上传到 GitHub。

### 广东现货系统账号

复制示例配置：

```bash
cp "上网电量抓取/config.example.json" "上网电量抓取/config.json"
cp "每日生产经营情况汇报自动生成工具/scripts/config.example.json" \
   "每日生产经营情况汇报自动生成工具/scripts/config.json"
```

编辑生成的 `config.json`，按示例填写 `username`、`password` 等字段。

下列工具各自保存独立登录态，可以并行运行，互不占用同一个浏览器 profile：

```text
上网电量抓取/auth_state.json
市场表更新/auth_state.json
集团每日上传/auth_state.json
每日生产经营情况汇报自动生成工具/scripts/auth_state.json
```

第一次使用“导出上网电量”“市场表更新”或“上传集团每日数据”时：

1. 打开对应功能页。
2. 点击“登录/刷新登录状态”。
3. 在弹出的浏览器中完成登录和必要的验证码操作。
4. 等待工具保存登录状态后关闭浏览器。
5. 回到工具箱执行正式任务。

登录失效时重复以上步骤即可。不要把某个工具的 `auth_state.json` 或 `.browser-profile/` 复制给另一个工具。

### 集团每日上传配置

集团每日上传默认读取 `上网电量抓取/config.json` 中的账号密码，但使用自己独立的登录态。需要独立账号时，可在以下位置创建同结构配置：

```text
集团每日上传/config.json
```

### 广东电价预测配置

广东电价预测默认复用 `上网电量抓取/config.json`，界面不要求再次输入账号密码。需要独立配置时执行：

```bash
cp "广东电价预测/config.example.json" "广东电价预测/config.json"
```

配置中的关键路径：

- `masterWorkbook`：长期维护的唯一数据总表，也是默认建模输入。
- `initialWorkbook`：只在首次创建总表时导入历史数据，后续不会替代总表。

### 私有数据上传配置

```bash
cp "电力工具脚本/private-data-uploader-tool/upload.config.example.json" \
   "电力工具脚本/private-data-uploader-tool/upload.config.json"
```

根据示例填写上传地址、匹配规则和浏览器配置。macOS 上如需显式指定 Chrome：

```text
/Applications/Google Chrome.app/Contents/MacOS/Google Chrome
```

### WPS/KDocs 配置

WPS 写入工具会在以下位置保存本机配置、登录 profile 和日志：

```text
wps自动/wps_excel_to_kdocs_config.json
wps自动/wps-browser-profile/
wps自动/logs/
```

首次写入时按界面提示登录 WPS/KDocs。后续只要 profile 有效，通常无需重复登录。配置可在工具界面中导入或导出。

## 功能使用说明

日期统一使用 `YYYY-MM-DD`，例如 `2026-07-16`。界面中的 `D` 表示所填写的基准运行日。

### 1. 导出上网电量

用途：按日期抓取各公司、各机组上网电量并生成 Excel 汇总。

操作步骤：

1. 首次使用先点击“登录/刷新登录状态”。
2. 在“运行日期”中填写需要导出的业务日期；默认值为昨天。
3. 点击“开始导出”。
4. 在页面日志中确认抓取和文件保存结果。
5. 点击“打开输出目录”查看 Excel。

输出目录：`上网电量抓取/输出/`。

### 2. 市场表更新

用途：按一个基准运行日抓取多类市场数据，写入指定的年度市场 Excel。该功能会修改所选工作簿，运行前请关闭 Excel/WPS，并建议先保留备份。

操作步骤：

1. 首次使用点击“登录/刷新登录状态”。
2. 填写“基准运行日期”。
3. 选择市场 Excel；默认路径为桌面日常工作目录中的 `现货统调市场出清、结算数据2026.xlsx`。
4. 确认工作簿包含 `2026年市场情况` 和 `2026年运行方式及成本` 工作表及对应日期行。
5. 关闭正在打开该文件的 Excel/WPS 窗口。
6. 点击“开始更新”，等待完成提示。

当前写入口径：

| 数据 | 来源日期 | 目标工作表和列 |
| --- | --- | --- |
| 市场负荷信息：日前 | D | `2026年市场情况`，`B:U` |
| 日前市场 12 项 | D-1 | `2026年市场情况`，`V:AG` |
| 市场负荷信息：实际 | D-2 | `2026年市场情况`，`AH:BA` |
| 实时市场 4 项 | D-2 | `2026年市场情况`，`BB:BE` |
| 发电侧结算 12 项 | D-6 | `2026年市场情况`，`BF:BQ` |
| 用电侧结算 7 项 | D-6 | `2026年市场情况`，`BR:BX` |
| 单位变动成本对应电价 | D | `2026年运行方式及成本`，`N:X` |
| 在线机组数、预测检修容量 | D | `2026年运行方式及成本`，`AA`、`AE` |
| 燃煤/燃气开机台数 | D | `2026年运行方式及成本`，`AB:AC` |
| 机组运行方式 | 复制 D-1 | `2026年运行方式及成本`，D 行 `B:M` |

如果某一数据源当天不可用，日志会说明该映射被跳过；其他可用映射仍会继续处理。

### 3. 广东电价预测

用途：补齐广东现货历史数据总表，并使用滚动训练生成 D+1 日前电价预测。

补齐数据总表：

1. 确认账号配置已完成。
2. 填写“运行日 D”。
3. 点击“补齐数据总表”。
4. 工具检查截至 D 日缺少 96 个十五分钟时点的日期，只补抓空缺数据。
5. 数据持续写入唯一的 `广东电价数据总表.xlsx`。

运行预测：

1. 先确保数据总表已存在并包含足够历史数据。
2. 选择预测输出目录。
3. 设置向后预测天数，默认 1 天，可选 1 至 7 天。
4. 设置滚动训练月数，默认最近 3 个自然月，最少 2 个月。
5. 点击“运行 D+1 预测”。

建模会自动排除实际/实时等会造成未来信息泄漏的字段。默认输出：

- `广东电价预测/输出/广东电价数据总表.xlsx`
- `广东电价预测/输出/D+1预测/D+1日前电价.xlsx`
- `广东电价预测/输出/D+1预测/MAE.json`

预测 Excel 包含 D+1 日期、96 个十五分钟时点和日前电价；`MAE.json` 保存模型误差指标。

### 4. 电力交易分析

用途：读取一个或多个“出清情况”Excel，批量生成 HTML 分析报告。

操作步骤：

1. 点击“添加文件”选择一个或多个 `.xlsx`；也可以点击“自动匹配D和D-2”。
2. 自动匹配会在 `~/Downloads` 中查找文件名含“出清情况”的 D 和 D-2 文件，并选择每个日期最后修改的一份。
3. 使用“移除选中”或“清空列表”调整文件列表。
4. 选择输出目录，默认是 `~/Downloads`。
5. 点击“开始分析”。

每个输入文件会生成一个 `<原文件名>_分析报告.html`。

### 5. 电量汇总

用途：汇总复盘目录中的日前和实时交易结果，生成 Excel，并尽量保留原表显示精度。

操作步骤：

1. 选择“原始数据文件夹”。界面会尝试填写当天默认复盘目录。
2. 确认“输出文件”；默认是原始目录下的 `<文件夹名>_汇总.xlsx`。
3. 点击“开始汇总”。
4. 完成后点击“打开输出目录”检查结果。

建议不要把输出文件放到另一个正在被汇总扫描的输入目录中，以免后续重复读取。

### 6. 生成报告

用途：抓取每日网站数据并基于 3 个 Excel 数据源生成生产经营情况 Word 报告。

数据日期口径：

- 前一日 Excel：报告日 D-1 的上网电量。
- 报告日 Excel：报告日 D 的日前电量/价格。
- 前两日 Excel：报告日 D-2 的日清数据。

操作步骤：

1. 填写“报告日期”。
2. 点击“按日期自动匹配 Excel”，从 `~/Downloads` 匹配文件名含“出清情况”的 D-1、D 和 D-2 文件；也可手动逐一选择。
3. 如需使用指定格式，选择 Word 模板；留空时使用脚本默认模板。
4. 根据需要勾选“抓取网站数据并汇总 Excel”和“生成 Word 报告”，至少选择一项。
5. 点击“开始生成”。两项都勾选时会先抓取，再生成 Word。

生成 Word 时 3 个 Excel 均为必填。输出目录：`每日生产经营情况汇报自动生成工具/输出/`。

### 7. 私有数据上传

用途：按配置规则匹配复盘目录中的文件，并上传到私有数据服务。

操作步骤：

1. 选择复盘文件夹。
2. 点击“预览”，核对日志中的文件匹配结果和上传目标。
3. 确认预览无误后点击“开始上传”。
4. 在确认对话框中再次核对目录，然后执行上传。

上传属于外部写入，不会自动重试。若上传中断，应先在目标系统确认已成功的文件，再决定是否重新执行。

### 8. 上传集团每日数据

用途：上传能销和省内日报数据；上传完成后自动新建集团日报。

操作步骤：

1. 首次使用点击“登录/刷新登录状态”。
2. 工具启动时会在 `集团每日上传/` 中自动匹配 D-2 的最新能销 Excel。
3. 如需更换文件，点击“选择文件”。可以只选择能销数据，也可以同时选择省内和能销数据。
4. 确认界面识别出的日期、数据类型和目标省份为“广东省”。
5. 正常情况下保持“已上传数据也覆盖”未勾选；只有明确需要覆盖目标数据时才勾选。
6. 点击“开始上传”并确认文件清单。

上传和日报创建属于外部写入，任务开始后不可从工具箱安全取消。不要重复点击或同时启动相同日期的上传。

### 9. WPS 写入工具

用途：把本地 Excel 或 KDocs 来源区域写入一个或多个目标 KDocs/WPS 表格区域。

基本流程：

1. 新建或选择一个文档配置。
2. 填写目标 KDocs URL。
3. 选择来源类型：本地 Excel 或 KDocs。
4. 本地 Excel 来源需选择文件；KDocs 来源需填写来源 URL。
5. 为每个区域填写来源工作表、来源起止单元格、目标工作表和目标起止单元格。
6. 确认来源区域和目标区域行列数一致。
7. 首次运行时在打开的 Chrome/WPS 页面中完成登录。
8. 执行写入并查看页面日志；需要复用配置时使用导入/导出功能。

写入前建议先在目标表保留副本。不要同时用多个窗口写入同一个目标区域。

## 后台任务与取消

耗时任务在后台运行，切换功能页不会中断任务。右侧任务中心显示任务状态和日志。

- 数据读取、预览和报告生成等任务在安全时可以取消。
- 上传、覆盖 Excel 和外部写入不会自动重试，部分任务也不可取消。
- 关闭工具箱时，如果仍有任务运行，程序会列出任务并要求确认。
- 工具箱只终止由本次工具箱启动的线程或子进程，不会关闭其他独立运行的程序。

同一个 Excel 正在被 Excel/WPS 打开时，写入工具可能报权限错误。先关闭工作簿，再重新运行。

## 迁移到另一台 Mac

推荐使用 Git 克隆最新版本。如果需要离线迁移，也可以复制整个项目文件夹，但不要依赖旧电脑的 `.venv`：

1. 把项目复制到新 Mac。
2. 可以删除复制来的 `.venv`，由新电脑重新创建。
3. 在项目根目录执行隔离属性和执行权限命令。
4. 运行 `./setup_macos.command`。
5. 按“首次登录和本地配置”重新准备配置与登录态。
6. 运行启动 smoke test，再启动图形界面。

本地账号配置可以通过安全方式单独迁移，但浏览器 profile 和登录态可能与旧电脑环境绑定；遇到失效时应在新电脑重新登录。

## 代理和网络

脚本会读取常见代理环境变量。例如本机代理端口为 `7897` 时：

```bash
export HTTPS_PROXY=http://127.0.0.1:7897
export HTTP_PROXY=http://127.0.0.1:7897
```

如果代理端口不同，请按实际设置修改。遇到登录页面打不开、Playwright 超时或上传请求失败时：

1. 先用 Chrome 手动确认目标系统可以访问。
2. 检查 VPN、代理软件和代理端口是否正常。
3. 重新执行对应页面的“登录/刷新登录状态”。
4. 查看页面中最早出现的错误，不要只根据最后一行判断原因。

## 常见问题

### 双击 `.command` 没有反应

在项目根目录执行：

```bash
xattr -dr com.apple.quarantine .
chmod +x setup_macos.command "电力工具箱.command" "00_启动/电力工具箱.command"
./电力工具箱.command
```

终端会保留具体错误信息，便于定位安装或路径问题。

### 提示缺少 Python 模块

重新安装根目录依赖：

```bash
./setup_macos.command
```

确认测试或脚本使用 `.venv/bin/python`，不要混用另一个 Python 环境。

### Playwright 找不到浏览器

```bash
.venv/bin/python -m playwright install chromium
```

### 登录状态失效或页面跳回登录页
